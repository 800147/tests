# Бэк для notifications

Это бэкенд для <https://800147.github.io/tests/notifications/>.
Он нужен для того, чтобы запросы к push-серверу шли с бэка,
а не из браузера, потому что при обращении к push-серверу
из браузера, возникает ошибка CORS (кроме сервера Фаерфокса —
он принимает запросы и из браузера тоже).

По-хорошему, на бэке должно быть больше логики, например,
генерация VAPID-ключей и связь их с пользователем, но здесь
такого нет. Изначально я планировал обойтись вообще без бэка
и оно даже работает без бэка для Фаерфокса, но так как другие
браузеры требуют бэк, был сделан этот простой костыль. Всё,
что делает это приложение — отправляет запрос к push-серверу
ровно с теми данными, которые пришли с фронта. Для демки
этого достаточно, но при настоящей реализации, конечно же,
надо генерацию VAPID ключей убрать с фронта и секретным ключом
оперировать только с бэка.

## Инструкция запуска

Внутри папки `notifications-sender-back` выполнить:
```
npm i
npm run dev
```
